# Public Release Checklist

Use this checklist before publishing any asset derived from Andrew OS, Kindora OS, private conversations, or client work.

## Confidentiality

- [ ] Contains no private client, employer, partner, donor, or opportunity information.
- [ ] Contains no private relationship notes, contact information, or personal identifiers.
- [ ] Contains no unpublished pricing, compensation, negotiation, or commercial strategy.
- [ ] Contains no protected health, employment, legal, or financial information.
- [ ] Contains no private repository paths, screenshots, access details, or source exports.

## Security

- [ ] Contains no API keys, passwords, tokens, cookies, certificates, private keys, or environment values.
- [ ] Contains no operational infrastructure details that create avoidable security risk.
- [ ] Commit history has been checked for accidentally introduced secrets.

## Intellectual property

- [ ] Andrew owns the material or has permission to release it.
- [ ] Third-party content is properly attributed and licensed.
- [ ] No proprietary proposal, presentation, report, or client language is reproduced without authorization.
- [ ] Public release does not undermine future commercial use or patentability.

## Data and evidence

- [ ] Data are synthetic, anonymized, or openly licensed.
- [ ] Sources are verifiable and citations are accurate.
- [ ] AI-generated claims and references have been checked.
- [ ] Results can be reproduced from the materials provided.

## Positioning

- [ ] The asset reflects Andrew's intended public professional positioning.
- [ ] Draft reasoning, speculation, or unresolved claims are labeled clearly.
- [ ] The asset does not imply endorsement by prior employers, clients, or partners.

## Release decision

- [ ] Approved for public release.
- [ ] License selected.
- [ ] Destination folder selected.
- [ ] Version and status updated.
- [ ] Build Ledger entry added.
