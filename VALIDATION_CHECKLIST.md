# Andrew Lab Validation Checklist

## Public safety

- [ ] Repository remains intentionally public.
- [ ] No private-source export has been committed.
- [ ] No confidential, identifying, or commercially sensitive information is present.
- [ ] No secrets or credentials are present in files or commit history.
- [ ] Data are synthetic, anonymized, or openly licensed.

## Asset quality

- [ ] Purpose or hypothesis is clear.
- [ ] Inputs and provenance are documented.
- [ ] Method and reproduction steps are present.
- [ ] Limitations and risks are stated.
- [ ] AI-generated claims and citations have been verified.
- [ ] License and attribution obligations have been addressed.

## Governance

- [ ] Lifecycle status is correct.
- [ ] Public Release Checklist is complete when the asset originated elsewhere.
- [ ] Build Ledger is updated.
- [ ] Graduation destination, if any, is identified.
- [ ] Explicit stage approval exists before advancing the repository build.
