# Andrew Lab

Andrew Lab is the public experimental repository within the Executive Digital Twin architecture.

It is a sandbox for developing, testing, documenting, and sharing public-safe ideas before selected assets graduate into the private Andrew OS or Kindora OS repositories.

## Repository role

Andrew Lab may contain:

- generic AI prompts and prompt experiments;
- prototype agents and workflows;
- synthetic-data demonstrations;
- open research notes and public evidence syntheses;
- reusable templates intended for public release;
- nonconfidential technical prototypes;
- public thought-leadership drafts;
- open-source utilities and methods.

Andrew Lab must not contain:

- raw ChatGPT, Claude, Gemini, email, or Notion exports;
- personal records or private career materials;
- client, partner, employee, or relationship intelligence;
- confidential proposals, contracts, financial information, or source documents;
- credentials, tokens, private keys, or authentication data;
- proprietary frameworks that have not been cleared for public release;
- protected health information or personally identifiable information.

## Relationship to the operating system

| Repository | Role | Visibility |
|---|---|---|
| Andrew OS | Canonical executive knowledge and intellectual capital | Private |
| Kindora OS | Consulting, career, business-development, and delivery operations | Private |
| Andrew Lab | Public-safe experimentation and publication staging | Public |

## Current stage

**Stage 0: Repository initialization**

The repository architecture, governance controls, experiment template, release checklist, and Build Ledger are being established. No compiler or private knowledge archive is included.

## Working areas

- `experiments/` — bounded tests and prototypes.
- `prompts/` — public-safe reusable prompts.
- `agents/` — prototype agent specifications.
- `research/` — open-source notes and evidence syntheses.
- `prototypes/` — code or workflow demonstrations.
- `publications/` — material being prepared for public release.
- `templates/` — standard experiment and publication templates.
- `governance/` — repository rules and release safeguards.
- `build-ledger/` — staged build and decision record.

## Governing rule

Nothing enters Andrew Lab merely because it is useful. It enters only when it is safe, lawful, nonconfidential, and intentionally suitable for public access.
